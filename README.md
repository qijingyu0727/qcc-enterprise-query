# QCC Enterprise Query

基于企查查公开接口的企业查询 skill，当前封装了 3 类能力：

- `1. 企业工商信息`：查询基础工商主体信息，成本更省
- `2. 企业信息核验`：查询增强核验字段，信息更完整，但费用更高
- `3. 企业模糊搜索`：按简称、电话、地址、人名、产品名、经营范围等线索找企业

## 快速用法

### 1. 首次使用先配置凭证

```bash
python3 scripts/manage_credentials.py --app-key "你的QCC_KEY" --secret-key "你的QCC_SECRET_KEY"
```

查看当前凭证状态：

```bash
python3 scripts/manage_credentials.py
```

凭证会写入 `qcc-enterprise-query/.env`，后续复用。

### 2. 常见查询方式

自动路由查询：

```bash
python3 scripts/run_query.py --company-name "企查查科技股份有限公司" --request "查企业信息"
```

查基础工商信息：

```bash
python3 scripts/registration_details.py --company-name "企查查科技股份有限公司"
```

查增强核验信息：

```bash
python3 scripts/basic_details.py --company-name "企查查科技股份有限公司"
```

按线索模糊搜索：

```bash
python3 scripts/fuzzy_search.py --search-key "010-62621818"
```

## 当前默认交互

- 只给企业全称或统一社会信用代码，且诉求只是“查企业信息 / 查基本信息”：
  默认直接返回 `1. 企业工商信息`
- 只给简称、电话、地址等线索：
  先走企业模糊搜索；用户确认企业后，如果原始诉求是泛查询，默认直接返回 `1. 企业工商信息`；如果原始诉求已经明确要查增强信息，则先提示费用，用户确认后再继续查 `2. 企业信息核验`
- 返回基础结果后：
  会继续提示用户，如果还想补 `人员规模、参保人数、行业、联系方式` 等信息，可以直接回复 `需要`，但这类继续查询可能会消耗较高费用
- 用户回复 `需要`：
  继续查询 `2. 企业信息核验`
- 用户一开始就明确要增强字段：
  继续查询 `2. 企业信息核验`
- 只要即将执行 `2. 企业信息核验`：
  如果是一开始就明确要求增强字段，仍会先提示费用更高

## 企业信息核验的费用提示

如果用户一开始就明确要求增强字段，建议使用下面这组话术：

- `你选的是企业信息核验。`
- `这个查询费用较高。`
- `如果确认继续，请直接回复 确认。`
- `如果无需查询参保人数、邮箱电话等信息，可以选择基础工商信息，回复 企业工商信息 或 1即可。`

## 适合怎么问

- `查一下“杭州飞致云信息科技有限公司”的企业信息`
- `查一下“杭州飞致云信息科技有限公司”的人员规模和参保人数`
- `查一下“小米”`
- `通过电话“010-62621818”找企业`

## 交互顺序

1. 企业名称不完整时，先走企业模糊搜索返回候选企业，只让用户确认企业全称。
   候选列表要带序号，用户可以直接回复企业全称，或者回复 `第几个/序号`。
2. 如果原始诉求只是“查企业信息 / 查基本信息”，企业确认后默认直接返回 `1. 企业工商信息`。
3. 基础结果返回后，再提示：
   如果还想补人员规模、参保人数、行业、联系方式等信息，可以直接回复 `需要`，但这类继续查询可能会消耗较高费用。
4. 用户回复 `需要` 时，直接继续查询 `2. 企业信息核验`；如果一开始就明确要增强字段，再先提示费用。
5. 查询完成后的后续建议，只能继续推荐本 skill 已支持的内容。
   不要提及股东信息、对外投资、司法风险、历史变更等未封装能力。
6. 用户可见输出里不要出现 `410 / 2001 / 886 / ApiCode / 脚本名` 等内部技术信息。
