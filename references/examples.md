# Examples

## 能力询问

```text
这个 skill 能干嘛？
```

期望行为：

- 不调用任何接口
- 直接介绍 3 种能力分别能查什么

## 精确主体详情

```text
用 $qcc-enterprise-query 查“企查查科技股份有限公司”的企业信息。
```

期望行为：

- 默认直接返回第 `1` 种企业工商信息
- 查完后再提示：如果还想补人员规模、参保人数、行业、联系方式等信息，可以直接回复 `需要`，但继续查询可能会消耗较高费用
- 不要先让用户做 `1/2` 选择

## 定向字段查询

```text
用 $qcc-enterprise-query 查“企查查科技股份有限公司”的参保人数和联系信息。
```

期望行为：

- 直接进入第 `2` 种费用确认
- 费用确认文案使用 `确认`
- 用户确认后再调用 `basic_details.py`
- 返回精简报告，只展示主体识别字段和请求字段

## 泛关键词模糊搜索

```text
用 $qcc-enterprise-query 通过电话“010-62621818”找企业。
```

期望行为：

- 调用 `fuzzy_search.py`
- 返回候选企业列表
- 候选企业列表带序号
- 提示用户可以直接回复企业全称或者第几个

## 简称 + 明确要增强信息

```text
查一下飞致云的企业信息核验
```

期望行为：

- 先调用 `fuzzy_search.py`
- 返回候选企业列表并要求用户确认具体企业
- 用户确认企业后，先提示这类查询费用较高
- 用户回复 `确认` 后，再返回企业信息核验结果
- 不要先返回企业工商信息

## 线索 + 明确要增强信息

```text
通过电话“0571-00000000”找企业，并查联系方式
```

期望行为：

- 先调用 `fuzzy_search.py`
- 返回候选企业列表并要求用户确认具体企业
- 用户确认企业后，先提示这类查询费用较高
- 用户回复 `确认` 后，再返回企业信息核验结果
- 不要再额外展示内部接口编号或脚本名

## 主体详情失败后回退

```text
用 $qcc-enterprise-query 查“企查查科技”的企业信息。
```

期望行为：

- 先调 `fuzzy_search.py`
- 返回候选企业列表
- 候选企业列表带序号
- 先让用户确认具体是哪一家
- 如果原始诉求只是泛查询，用户确认后默认直接返回第 `1` 种结果
- 基础结果返回后，再提示如需更多信息可以回复 `需要`，并说明继续查询可能会消耗较高费用
- 如果用户回复 `需要`，直接继续查第 `2` 种结果

## 查完后的追问边界

```text
查下杭州飞致云信息科技有限公司基本信息
```

期望行为：

- 默认直接返回第 `1` 种结果
- 后续提示用户如需更多信息可以回复 `需要`
- 用户若在这一步回复 `需要`，直接继续查第 `2` 种增强信息
- 如果要补一句“还可以继续查什么”，只能继续问：
  - 是否要按电话、地址、人名、产品名、经营范围等线索继续模糊搜索
- 也可以继续指定查询参保人数、联系信息、企业规模等核验字段
- 也可以建议继续查询其他企业，例如“小米科技有限责任公司”
- 不要主动推荐未封装能力

## 简称先查主体再确认

```text
查下杭州飞致云
```

期望行为：

- 先调用 `fuzzy_search.py`
- 返回候选企业并要求用户确认具体企业全称
- 提示语要明确支持“回企业全称或者第几个”
- 用户确认企业后，默认先返回第 `1` 种基础结果
