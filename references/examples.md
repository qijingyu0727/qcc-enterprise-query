# Examples

## 能力询问

```text
这个 skill 能干嘛？
```

期望行为：

- 不调用任何接口
- 直接介绍 3 个接口分别能查什么

## 精确主体详情

```text
用 $qcc-enterprise-query 查“企查查科技股份有限公司”的企业信息。
```

期望行为：

- 先解释 `1` 和 `2` 的区别
- 推荐第 `1` 种
- 等用户选择后再继续调用详情接口

## 定向字段查询

```text
用 $qcc-enterprise-query 查“企查查科技股份有限公司”的参保人数和联系信息。
```

期望行为：

- 先返回 `1/2` 选择说明
- 明确推荐第 `2` 种
- 讲清第 `1` 种包含哪些工商信息，以及第 `2` 种额外多出哪些增强字段
- 用户选中第 `2` 种后，再进入费用确认
- 费用确认文案使用 `确认`，不要要求用户回复 `确认查询`
- 用户确认后再调用 `basic_details.py`
- 返回精简报告，只展示主体识别字段和请求字段

## 泛关键词模糊搜索

```text
用 $qcc-enterprise-query 通过电话“010-62621818”找企业。
```

期望行为：

- 调用 `fuzzy_search.py`
- 返回候选企业列表

## 主体详情失败后回退

```text
用 $qcc-enterprise-query 查“企查查科技”的企业信息。
```

期望行为：

- 先调 `fuzzy_search.py`
- 返回候选企业列表
- 候选企业列表带序号
- 先让用户确认具体是哪一家
- 用户确认后，再单独进入 `1/2` 选择
- 这一轮要明确讲清第 `1` 种和第 `2` 种的字段差异
- 如果最终走第 `2` 种，还要先经过费用确认

## 查完后的追问边界

```text
查下杭州飞致云信息科技有限公司基本信息
```

期望行为：

- 先返回 `1/2` 选择说明
- 默认推荐第 `1` 种
- 用户若改选第 `2` 种，还要先经过费用确认
- 如果要补一句“还可以继续查什么”，只能继续问：
  - 是否要按电话、地址、人名、产品名、经营范围等线索继续模糊搜索
- 也可以继续指定查询参保人数、联系信息、企业规模等核验字段
- 也可以建议继续查询其他企业，例如“小米科技有限责任公司”
- 不要主动推荐未封装能力

## 简称先查主体再确认

```text
查下杭州飞致云
```

期望行为：

- 先调用 `fuzzy_search.py`
- 返回候选企业并要求用户确认具体企业全称
- 提示语要明确支持“回企业全称或者第几个”
- 不要直接走已经移除的企业名称搜索接口
- 不要在候选企业确认这一步同时询问用户想查第 `1` 种还是第 `2` 种
